# Getting Started

Get your Expo Open OTA server running in under a minute.

## Prerequisites

Before running the server, you'll need:

- **Docker** installed on your machine
- An **Expo Access Token** — [How to get one](/docs/prerequisites#how-to-get-your-expo-token)
- Your **Expo App ID** — [How to find it](/docs/prerequisites#how-to-get-your-project-id)

## Run the server

```bash
docker run -p 3000:3000 \
  -v ./updates:/updates \
  -e STORAGE_MODE=local \
  -e BASE_URL=http://localhost:3000 \
  -e EXPO_ACCESS_TOKEN=your_token \
  -e EXPO_APP_ID=your_app_id \
  -e JWT_SECRET=your_secret \
  -e ADMIN_PASSWORD=your_password \
  -e USE_DASHBOARD=true \
  ghcr.io/axelmarciano/expo-open-ota:latest
```

That's it. Your OTA server is now running at [http://localhost:3000](http://localhost:3000) with the dashboard enabled at [http://localhost:3000/dashboard](http://localhost:3000/dashboard/).

### What each variable does

| Variable | Description |
| --- | --- |
| `STORAGE_MODE=local` | Store update assets on the local filesystem |
| `BASE_URL` | The public URL of your server (used to generate asset URLs) |
| `EXPO_ACCESS_TOKEN` | Authenticates with the Expo API ([get yours here](/docs/prerequisites#how-to-get-your-expo-token)) |
| `EXPO_APP_ID` | Your Expo project ID ([find it here](/docs/prerequisites#how-to-get-your-project-id)) |
| `JWT_SECRET` | Secret used to sign JWT tokens for secure endpoints |
| `ADMIN_PASSWORD` | Password for the dashboard login |
| `USE_DASHBOARD` | Enables the web dashboard |

The `-v ./updates:/updates` flag mounts a local directory so your updates persist across container restarts.

## Going further

This minimal setup is great for testing. For production, consider:

- **[S3 Storage](/docs/storage?storage=s3)** — Reliable, scalable, and supports CDN integration
- **[CloudFront CDN](/docs/cdn/cloudfront)** — Serve assets faster with signed URLs
- **[Redis Cache](/docs/cache?cache=redis)** — Optional caching layer for better performance with multiple instances

See the full [Environment Variables](/docs/environment) reference for all configuration options.

## Deploy to production

When you're ready to go live:

- **[Railway](/docs/deployment/railway)** (recommended) — One-click deploy, HTTPS and scaling out of the box
- **[Docker](/docs/deployment/custom)** — Run the image on any infrastructure
- **[Helm](/docs/deployment/helm)** — Deploy on Kubernetes with a single command
